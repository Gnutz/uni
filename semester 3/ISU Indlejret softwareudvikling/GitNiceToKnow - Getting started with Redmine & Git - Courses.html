Error when pushing large data set to git server¶

The error would look something like this:

Counting objects: 5, done.
Delta compression using up to 4 threads.
Compressing objects: 100% (4/4), done.
error: RPC failed; result=22, HTTP code = 404
fatal: The remote end hung up unexpectedly
Writing objects: 100% (4/4), 1.34 MiB | 0 bytes/s, done.
Total 4 (delta 0), reused 0 (delta 0)
fatal: The remote end hung up unexpectedly
Everything up-to-date

Note especially the RPC failed; ....

If you are using SmartGit (Windows/Linux) you may get:

The problem is that there is a default limit as to how much data may be pushed to the server. Running the following command raises this to the amount specified.
For Linux the solution is do the following in a terminal:

git config --global http.postBuffer 524288000

By adding option --global it applies to all repositories...

In Windows you should edit the file: C:\Program Files (x86)\SmartGitHg 4.6\git\etc\gitconfig.
At this point it is presumed that you installed SmartGitHg at the above shown location, otherwise you would have to adjust the specified path.

Below is a gitconfig file that has been adjusted and thus works.

Alternately, find the program bash that is found in the same subdir as git assuming that you are using SmartGit. Run this program and run the above mentioned command. It will work as expected and thus like it does in Linux.

SSL certificate error¶

For self signed certificates curl will issue an error and thus git will terminate.
Run the following command to have curl ignore the issue.

For Linux run the following command line in a terminal in the git repo:

git config --global http.sslVerify false

Do note that the above command effectively ignores SSL certificate errors on any site that you will encounter, so in that respect it is a bit insecure!
In Windows the same applies, however you should run it within the program bash which resides in a subdir to SmartGit.
